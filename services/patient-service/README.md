# Patient Service

This is the Patient Management microservice for the Doctor365 platform.

## Overview
This service is responsible for managing patient profiles, updating demographics and contact details, and viewing initial medical history summaries.

It connects cleanly with the standard JWT authentication generated by `auth-service`.

## Setup and Run
1. Configure `.env` based on `.env.example` or team defaults (Requires `JWT_SECRET`, `PORT`, `MONGODB_URI`).
2. Run `npm install`
3. Running standalone: `npm start` or `npm run dev`
4. Running via docker: `docker-compose up --build patient-service` from root.

## API Endpoints

### 1. Health Check
* **Method:** `GET`
* **Route:** `/health`
* **Access:** Public
* **Response:**
  ```json
  {
    "status": "ok",
    "service": "patient-service"
  }
  ```

### 2. Get Current Patient Profile
* **Method:** `GET`
* **Route:** `/me`
* **Access:** Protected (patient, admin)
* **Response (Success):**
  ```json
  {
    "success": true,
    "message": "Patient profile retrieved successfully",
    "data": { ...patientProfile }
  }
  ```

### 3. Update/Complete Current Patient Profile
* **Method:** `PUT`
* **Route:** `/me`
* **Access:** Protected (patient, admin)
* **Payload Example:**
  ```json
  {
    "firstName": "John",
    "lastName": "Doe",
    "dateOfBirth": "1990-01-01",
    "gender": "male",
    "phone": "+94771234567",
    "address": "123 Colombo St",
    "emergencyContact": "+94777654321",
    "medicalHistorySummary": "No prior conditions"
  }
  ```

### 4. Get Patient by ID
* **Method:** `GET`
* **Route:** `/:id`
* **Access:** Protected (admin, doctor)
* **Note:** `:id` falls back to search by auth `userId` if the Mongo `_id` is not matched.

## Notes
- To test the API fully, checkout the included `patient-service-postman-collection.json` and import it into Postman.
- The `authMiddleware` acts dynamically and reads any valid standard Token without making synchronous calls to the auth-service logic.
